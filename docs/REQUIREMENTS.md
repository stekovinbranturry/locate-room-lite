# LocateRoom Lite — 需求说明

> 前端工程师 · 带回家小项目

可测试的验收项见 [验收清单](./ACCEPTANCE.md)。

## 项目概述

用 **WebRTC** 实现一个 Web 端的「实时位置共享小房间」：

1. 用户创建房间，生成分享链接
2. 好友通过链接加入同一房间
3. 大家在地图上看到彼此位置的实时更新

## 核心功能

### 房间

- 创建房间
- 通过链接加入
- 在线成员列表
- 加入 / 离开实时通知

### 位置共享

- 每人以 **10 Hz** 上报位置
- 在地图上实时展示
- **数据必须通过 WebRTC DataChannel P2P 传输**（位置数据不走服务端中转）

### 稳定性

- 新成员加入后，应**立即看到**所有人的当前位置
- 断网后应能**自动恢复**
- 能感知到其他人**掉线**

## 性能底线

| 指标 | 要求 |
|------|------|
| 房间规模 | 支持 **4 人** 房间 |
| 位置更新 | **10 Hz** |
| 渲染帧率 | **≥ 50 fps** |
| 首屏可见 | 加入房间到**首次看到他人位置** ≤ **3 秒** |
| 移动端 | 必须可用 **H5**（中端 Android + iOS Safari） |
